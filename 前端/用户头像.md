作为开发者，**用户头像的最佳方案是：上传 512×512px 正方形原图，前端按场景生成多套缩略图**。这是兼顾清晰度、兼容性、性能与未来扩展性的行业通用标准。

### 一、核心推荐（开发者视角）
#### 1. 上传尺寸（后端存储）
- **首选：512×512px（1:1 正方形）**
    - 适配所有主流平台（Web、App、小程序）
    - 缩放至 48px/96px/128px/256px 均清晰
    - 支持 Retina 高清屏（@2x/@3x）
- **备选：640×640px**
    - 适合需要极高清展示的场景（如个人主页大图）
    - 注意控制文件大小（建议 < 200KB）

#### 2. 前端展示尺寸（按场景）
| 场景 | 推荐尺寸 | 用途 |
| :--- | :--- | :--- |
| 列表/评论区 | **48×48px** | 消息列表、评论、侧边栏 |
| 普通头像 | **96×96px / 128×128px** | 个人资料、导航栏、卡片 |
| 主页大图 | **256×256px / 512×512px** | 个人主页、详情页 |
| 移动端适配 | 144×144px / 192×192px | Android/iOS 通知、小组件 |

### 二、技术实现要点
1. **必须正方形（1:1）**
    - 避免拉伸/裁切变形
    - 圆形头像时关键内容居中（安全区：中心 80% 区域）

2. **多倍图策略（Retina 适配）**
    - 提供 @1x（基础）、@2x（高清）、@3x（超高清）三套图
    - 示例：48px (@1x) → 96px (@2x) → 144px (@3x)

3. **格式与压缩**
    - 格式：**WebP > PNG-24 > JPG**（兼顾透明与体积）
    - 压缩：质量 80–90，文件大小控制在 **50–200KB**
    - 禁止上传过大原图（如 > 2048px），后端统一裁剪

4. **后端处理建议**
    - 接收用户上传后，自动裁剪为 512×512 基准图
    - 预生成各场景缩略图并缓存
    - 限制上传最小尺寸（如 ≥ 200×200px），避免模糊

### 三、主流平台参考（便于对接）
| 平台 | 推荐上传尺寸 | 显示尺寸 |
| :--- | :--- | :--- |
| 微信/公众号 | 640×640px | 132×132px / 240×240px |
| 抖音/小红书 | 400×400px | 80×80px / 160×160px |
| Discord | 512×512px | 128×128px |
| 通用 Web/App | 512×512px | 48–256px（按需） |

### 四、总结与行动
- **最佳实践**：后端存 **512×512px** 基准图，前端按需渲染 48/96/128/256px 缩略图。
- **关键**：正方形、居中、多倍图、合理压缩。
